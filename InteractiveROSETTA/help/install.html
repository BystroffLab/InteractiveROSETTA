InteractiveROSETTA - Installation

Windows 7/8

A Windows installer is provided in the "win_install" directory. Simply double click on "install.exe" to launch an installer that will guide you through the process.

Manual Installation

If the automatic installer fails, you can always install all the necessary components manually. First, you need a Python2.7 installation, which Windows does not have by default. We recommend using the Miniconda Python2.7 package because it contains a package manager called "conda" that makes installing 64bit Python modules simple. Miniconda can be downloaded and installed from here: http://conda.pydata.org/miniconda.html. Select the Python2.7 download. You can check whether you are running a 32 or 64bit Windows by going to Start Menu->Control Panel->System and Security->System and looking at the value for "System Type."

Next, you need to install some Python packages by using the conda package manager. This can be accomplished from the command line. Go to the Start Menu, type "cmd" in the search box, and hit ENTER to open a terminal. Type the following lines, followed by pressing ENTER to submit them:

conda install pip

conda install numpy

conda install poster

Enter "y" when asked if you really want to install these modules

Now you need to install wxPython, which is the module that draws all the graphics. It can be downloaded from here: http://www.wxpython.org/download.php. Again, you want the py27 version for either 32 or 64bit Windows, whichever you are running.

Next, you need to install some extra Python modules for BioPython, PyMOL, and OpenBabel. The links are here: http://www.lfd.uci.edu/~gohlke/pythonlibs/#biopython, http://www.lfd.uci.edu/~gohlke/pythonlibs/#pymol, and http://www.lfd.uci.edu/~gohlke/pythonlibs/#openbabel.

These are "wheel" files, and you need to download the appropriate file for Python2.7 (not 26, 33, or 34) and either the win32 or win_amd64 file. Save them to a place that is easy to access, such as the Downloads folder. You need to use pip to install these wheel files, so again open a command prompt by typing "cmd" in the search box of the Start Menu. Go to the directory where the wheel files are saved. Assuming you saved them to the Downloads folder:

cd Downloads

If you are having trouble getting to the directory, then open the folder to which you saved them and copy them to your home folder at C:\Users\Username since the command prompt puts you in this folder by default. Then install them by doing the following:

pip install biopython-1.65-cp27-none-win_amd64.whl

pip install pymol-1.7.2.1-cp27-none-win_amd64.whl

pip install openbabel-1.8.1-cp27-none-win_amd64.whl

The above assumes you are on a 64bit machine. Replace the filenames with the 32bit version if this is not the case.

Now you have to install OpenBabel proper, which installs some important files that the Python version of OpenBabel requires. You can download it here: http://openbabel.org/wiki/Category:Installation.

Finally, you need to get a copy of PyRosetta from the Gray lab. The download page is here: http://www.pyrosetta.org/dow. This requires obtaining an academic or commercial license.

Once all of this is completed, you can run InteractiveROSETTA by double clicking on the "InteractiveROSETTA.py" file in the main folder. Right-click on it and select the option to add a shortcut if you wish to have a link to it on your desktop or in the Start Menu.

Linux

A bash installer is provided to easily install all of the packages. Run it from a terminal to use it.

Manual Installation

To install manually, you need to have a Python2.7 install which can be installed from your package manager. Older versions of Python may work but they have not been tested. Make sure to install the following packages using pip or easy_install: biopython, psutil, poster, glob, and numpy.

Make sure that wxPython is installed. It is available in your package manager. For yum users, it is called "wxPython". For apt-get users, it is called "wxgtk-2.8" or something similar. Also, you need to install the open-source version of PyMOL which is also available in your package manager.

Installing OpenBabel is optional (it makes creating residues and ligands easier). For Debian distros, you can install it from apt-get. The package you need is called "python-openbabel". OpenBabel is available in yum but does not seem to install properly. So you need to compile it from source. This requires installing cmake and gcc. Download the source here: http://open-babel.readthedocs.org/en/latest/Installation/install.html#compiling-open-babel. Don't forget to add the -DPYTHON_BINDINGS flag to the cmake call or you won't get the Python bindings.

Finally, download PyRosetta from http://www.pyrosetta.org/dow. This requires obtaining an academic or commercial license. You can unpack the PyRosetta directory anywhere you like (/usr/local seems to be the default global location). There is a script in the main PyRosetta directory called "SetPyRosettaEnvironment.sh" that needs to be sourced before you use PyRosetta, so add a line to your .bashrc file to source that script.

If you are using the Ubuntu version, you need to change the name of the main PyRosetta directory to eliminate the ":", otherwise your PYTHONPATH environment variable will not be interpreted correctly. You also need to move the two boost_python SO files to the "rosetta" directory. The ScientificLinux PyRosetta package is fine the way it is.

If PyRosetta is installed in a root-privileged directory, run python as root and perform a simple scoring routine to generate the Dunbrack library binary file to greatly speed up future PyRosetta calculations. Do the following:

sudo python

>>> from rosetta import *

>>> init(extra_options="-ignore_unrecognized_res")

>>> pose = pose_from_pdb("/path/to/valid/pdbfile")

>>> scorefxn = create_score_function("talaris2013")

>>> scorefxn(pose)

>>> quit()

Now you can run InteractiveROSETTA by executing "python InteractiveROSETTA.py" from a terminal.
